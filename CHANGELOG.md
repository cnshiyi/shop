# 版本记录

## 2026-04-11

### 新增
- 新建 Django 项目骨架，并迁移商城、用户、充值、监控等核心模型
- 新增 `run.py` 作为 PyCharm 一键启动入口
- 新增 `aiogram` 机器人实现，覆盖购买商品、订单、充值、充值记录、地址监控、个人中心等流程
- 新增 TRON 扫描器，用于转账监控与订单 / 充值支付匹配
- 新增 TRON 资源巡检器，用于监控可用能量 / 带宽增加
- 新增 Redis 缓存层，用于站点配置、监控地址缓存、FSM 状态、多用户隔离
- 新增 Redis 每日临时统计，用于记录地址当天收入 / 支出 / 利润
- 新增资源通知详情按钮与交易通知详情按钮

### 调整
- 主菜单精简为 `购买商品` 与 `个人中心`
- `我的订单 / 充值余额 / 充值记录 / 地址监控` 已移动到 `👤 个人中心`
- 个人中心按钮布局调整为每行两个
- 监控详情中加入 `监控转账` 与 `监控资源` 开关
- 监控说明补充：资源监控仅在可用能量 / 带宽增加时通知，资源消耗不通知
- 扫描日志改为安静模式：10 分钟摘要 + 命中时详细日志
- 监控通知样式美化，增加表情、代码样式、手续费信息

### 修复
- 修复 Telegram 按钮 `BUTTON_DATA_INVALID`，将过长回调数据改为短 key
- 修复转账手续费显示为 `未知` 的问题，改为查询链上交易详情后显示
- 修复转出仅计入统计、不提醒的问题，现已支持单独 `支出提醒`
- 修复今日收入 / 支出 / 利润为单笔假值的问题，改为 Redis 当天真实累计
- 修复利润无法正确表现负数的问题，现支持亏损显示，例如 `-100 USDT`
- 修复多用户状态混用风险，FSM 改为优先使用 `RedisStorage`

### 当前通知逻辑
- 转入：发送 `🟢 收入提醒`
- 转出：发送 `🔴 支出提醒`
- 资源增加：发送 `⚡ 资源变动提醒`
- 每条通知均支持详情查看

### 说明
- 每次重要代码变更均已提交到 Git
- 涉及模型字段新增时，需要执行数据库迁移
